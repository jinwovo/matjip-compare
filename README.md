# 🍚 맛집비교 지도 (good-restaurant)

지도에서 음식점을 누르면 **네이버·카카오·구글**의 별점, 별점 개수, 리뷰수를
호텔 최저가 비교처럼 한 화면에서 비교해주는 개인용 맛집 지도.

> ⚠️ **개인용/학습용 프로토타입입니다.** 네이버·카카오는 리뷰 데이터를 위한 공식 API를
> 제공하지 않아, 이 앱은 두 서비스의 웹 응답을 조회해 화면에 보여줍니다. 이 방식은 각 사
> 이용약관에 어긋날 수 있으므로 **스토어 배포·상업적 이용은 하지 마세요.** 출시가 목표라면
> 네이버·카카오는 "앱에서 열기" 딥링크로 대체하고, 별점 비교는 구글 공식 API만 사용하세요.

## 빠른 시작

```bash
npm install
npm start
```

브라우저에서 **http://localhost:5178** 접속 → 검색창에 `강남역 초밥` 같은 걸 입력 →
지도의 핀(또는 오른쪽 목록)을 클릭하면 3사 비교 카드가 뜹니다.
목록은 **추천순 / 평점순 / 리뷰순**으로 정렬할 수 있고, 상단 📍 버튼으로 현재 위치로 지도를 옮깁니다.

## 📱 앱으로 설치 (PWA)

이 앱은 PWA라서 브라우저 없이 네이티브 앱처럼 설치해 쓸 수 있습니다 (모바일·데스크톱 코드 한 벌).

- **아이폰/안드로이드**: 모바일 브라우저로 접속 → 공유(또는 ⋮) 메뉴 → **"홈 화면에 추가"**
- **데스크톱(Chrome/Edge)**: 주소창 오른쪽의 **설치 아이콘(⊕)** 클릭 → "설치"

설치하면 전체화면 아이콘 앱으로 실행되고, 앱 화면·지도 타일이 캐시되어 다음 실행이 빨라집니다.
스토어 정식 배포가 필요해지면 이 웹앱을 그대로 Capacitor로 감싸면 됩니다.

## 구글 별점까지 보려면 (선택)

`.env.example`를 참고해 환경변수 `GOOGLE_MAPS_API_KEY`를 설정하고 실행하세요.
없어도 네이버·카카오 비교는 정상 동작합니다.

```bash
# PowerShell
$env:GOOGLE_MAPS_API_KEY="발급받은_키"; npm start
```

## 데이터 출처와 현황 (2026-07 확인)

| 플랫폼 | 별점 | 별점 개수 | 리뷰수 | 취득 방식 |
|---|---|---|---|---|
| 카카오 | ✅ `rating_average` | ✅ `rating_count` | ✅ `reviewCount` | 지도 검색 응답 |
| 네이버 | ✅ 방문자 리뷰 평점 | (미공개, 리뷰수로 대체) | ✅ 방문자/블로그 리뷰수 | 플레이스 페이지 |
| 구글 | ✅ `rating` | ✅ `userRatingCount` | ✅ `userRatingCount` | 공식 Places API |

## 구조

```
server/
  index.mjs            Express 서버 (API + 정적파일 + TTL 캐시)
  providers/
    kakao.mjs          카카오 지도 검색 (별점 포함)
    naver.mjs          네이버 검색 + 플레이스 별점/리뷰수
    google.mjs         구글 Places API (키 있을 때)
  match.mjs            플랫폼 간 "같은 가게" 매칭 (이름 유사도 + 좌표 거리)
  score.mjs            베이지안 종합 점수·추천 정렬 알고리즘
public/
  index.html app.js style.css      Leaflet 기반 반응형 웹앱
  manifest.webmanifest             PWA 매니페스트 (설치 정보)
  service-worker.js                오프라인·타일 캐싱 서비스워커
  icons/                           앱 아이콘 (192·512)
```

핵심 난제는 **같은 가게를 3사에서 매칭**하는 것. `match.mjs`가 상호 bigram 유사도와
Haversine 거리를 조합해 판별합니다. 매칭된 별점은 `score.mjs`가 **베이지안 평균**으로
보정(리뷰 적은 고득점을 신뢰하지 않음)한 뒤 리뷰 볼륨으로 통합해 종합 점수를 냅니다.
